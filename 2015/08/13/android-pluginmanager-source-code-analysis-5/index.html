ActivityClassGenerator简介

ActivityClassGenerator是体现Activity PluginManager插件sdk思路独特的地方。ActivityClassGenerator使用DexMaker动态生成PluginActivity，这个PluginActivty继承于插件中的源Activity，在生成时重载了源Activity的一些方法，替换了Activity的Context、AssetManager，从而实现不修改原插件apk的情况下实现动态加载apk的能力。

在生成dex文件时，当API < 11 用jar代替。

总结

至此，Andorid PluginManager主体功能就全部分析完了。

回头来总结一下Andorid PluginManager的加载插件的过程和原理：

  • 替换宿主Application的ClassLoader为FrameworkClassLoader
  • 解析插件apk中的AndroidManifest.xml文件，保存Activity、Service、Content Provider等信息到PluginInfo中
  • 生成PluginClassLoader保存到PluginInfo备用
  • 生成插件AssetManager、Resources保存到PluginInfo备用
  • 实例化插件Application，并替换Application的Context
  • 生成继承自源Activity的PluginActivity
    • 定义PluginActivity类
    • 重载attachBaseContext()方法，用PluginInfo缓存的对象替换Context、AssetManager、Resources
    • 重载onCreate()等生命周期方法
    • 重载getAssets()、getResource()方法
    • 重载startActivityForResult()
    • 重载getIntent()等其他方法
    • 生成dex文件
    • 通过PluginClassLoader加载生成的dex
